# Запуск пайплайна

Все команды выполняются из корня репозитория. Разовая настройка окружения:

```bash
pip install -r requirements.txt   # зависимости
pip install -e .                  # проект как пакет (импорты config/core/entrypoint)
```

Editable-установка (`-e .`) заменяет ручную правку `sys.path`: после неё все точки
входа импортируются напрямую из любого места.

---

## Способ 1. Интерактивное меню (для демонстрации)

Пошагово спрашивает целевую величину, метод, режим синтеза и его параметры,
затем запускает оценку и печатает таблицу.

```bash
python -m entrypoint.interactive
```

Диалог:

```
Целевая величина:
  1) СП 63.13330
  2) Руководство 1978
  3) обе
Методы (через запятую, пусто — все):
  1) Линейная регрессия      ← напр. "1" или "1,2"; пусто = все
Включить синтез образцов? (Y/n)
  образцов синтеза на профиль [15]:
Сохранить результаты в results/? (y/N)
```

После прогона можно повторить с другими параметрами. Числа берутся из
[`config/settings.py`](../config/settings.py) как значения по умолчанию.

## Способ 2. Фиксированный CLI (для скриптов и воспроизводимости)

Каждый метод — свой файл-обёртка в `entrypoint/single/`; в `entrypoint/group/` —
сводные прогоны. Например:

```bash
python entrypoint/single/line_regression.py    # один метод
python entrypoint/group/baseline.py             # все линейные методы сразу
```

Флаги (общие для всех обёрток, см. [`_common.py`](_common.py)):

| Флаг | Значение | По умолчанию |
|------|----------|--------------|
| `--target {SP63,RUK78,both}` | какую целевую величину считать | `both` |
| `--no-synth` | отключить синтез образцов | синтез включён |
| `--samples N` | образцов синтеза на профиль | из `settings.SYNTH` |
| `--export` | сохранить CSV + PNG в `results/` | не сохранять |

Примеры:

```bash
# только СП63, без синтеза
python entrypoint/single/line_regression.py --target SP63 --no-synth

# обе цели, 30 синтетических образцов на профиль, с выгрузкой
python entrypoint/single/line_regression.py --samples 30 --export
```

## Способ 3. Собранный бинарь (демо без Python)

Однофайловый исполняемый файл с зашитыми данными и зависимостями — для показа
на машине, где не развёрнуто окружение. На Windows → `.exe`, на Linux → ELF.

Сборка (нужен `pip install -r requirements-build.txt`):

```bash
pyinstaller beam_demo.spec
```

Результат — `dist/beam_demo.exe` (~90 МБ). Запуск двойным кликом или из консоли;
интерфейс — то же интерактивное меню (Способ 1). Экспорт пишется в папку
`results/` рядом с исполняемым файлом.

> Бинарь и папки `build/`, `dist/` не коммитятся (см. `.gitignore`) — собирается
> локально из спека.

## Результат

- Таблица `метод × метрики` по каждой целевой величине печатается в консоль.
- С `--export` / ответом «да» в меню: `results/metrics_<цель>_<время>.csv` и
  `results/plots_<цель>_<время>.png` (scatter Qэксп/Qпред + сравнение RMSE).

Ключевые метрики: `Qexp/Qpred_mean` (в идеале ≈ 1) и `CV`
`MAE`/`RMSE`/`R2` — для полноты. `overfit` = `R2_train − R2(LOGO)`
показывает переобучение.
